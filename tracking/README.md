# tracking/ — Canonical Data & Playtest Reports

**Purpose:** Long-lived data the game and agents depend on, plus playtest outputs.

---

## Canonical (Must Keep Updated)

| File | Purpose | Update when |
|------|---------|-------------|
| `BUGS_FIXED_LOG.md` | Bug history, fixes, seeds for repro | You fix a bug |
| `card_database.csv` | All cards (boons, worship, libations, artifacts) | You add/change a card |
| `BOON_SPREADSHEET.csv` | Boon categorization | You add a boon category |

Referenced by: SOUL.md, 3-automation.mdc, CONSOLIDATED_BOON_REFERENCE, CARD_METADATA_REFERENCE.

---

## Generated (Playtest Outputs)

| File | Generated by |
|------|--------------|
| `FIRST_ANTE_PLAYTEST_REPORT.md` | `npm run playtest:ante1` |
| `WINNING_PLAYTEST_REPORT.md` | `npm run playtest:winning` |

These are overwritten each run. Use for translator improvement, balance checks.

---

## Analysis Reports (Manual / Ad hoc)

| File | Purpose |
|------|---------|
| `PLAYTEST_REPORT.md` | General playtest notes |
| `BOON_PLAYTEST_REPORT.md` | Boon-specific playtest |
| `STRESS_TEST_REPORT.md` | Stress test results |
| `GAMEPLAY_ANALYSIS_REPORT.md` | Deep gameplay analysis |
| `GAME_TESTER_PLAYTEST_ANALYSIS.md` | External tester feedback |
| `HEALTH_STATUS_REPORT.md` | Project health snapshot |

These are historical/periodic. Not auto-generated.
